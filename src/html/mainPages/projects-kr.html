금융 정보 검토 시스템 2024

다른 DB에 저장된 거래금액 데이터들의 오차를 판독해 Spring Scheduler의 설정에 맞추어 실행 후 Slack 봇과 AWS S3버킷으로 보고서 업로드 기능 개발.

DynamoDB, PostgreSQL, 그리고 MSSql의 세개의 데이터 베이스의 매일 몇십만개의 금융 거래 내역을 조회 하여 Cursor Streaming 기법으로 금액 비교.

Inner Join, Temp table, range 파티셔닝을 통해 빠른 데이터 인덱싱 검색 활용.

Lazy DB 데이터 탐색을 이용해 서버 메모리 절약. Optimistic lock을 사용해 의도치 않은 데이터 값 변경과 오보도 방지.

Java Record와 jackson 아티팩트랄 활용하여 획일화된 csv 보고서 파일 구축.

SOLID principle, Generics, Marker(tagging) interface를 적극적으로 활용하여 다른 종류의 보고서도 중복 코드없이 확장이 용이하게 구현.

회사 프로젝트

상호 펀드 주문 시스템 2023-2024

상호펀드거래를 위한 코어 펀드 서비스 개발 (Event Driven 아키텍처 서비스).

Frontend의 서버에서 요청된 거래 요청을 AWS DynamoDB의 Streaming으로 트리거하여 Kafka Message 서비스로 받아 금융 거래를 처래하는 방식.

Kafka의 load-balancing 전략과 customized partitioning으로 짧은 시간에 발생한 다량의 주문을 지연과 오류없이 처리.

다른 서비스와의 소통을 S3버킷-Visual Cron과 Apache AMQ서비스로 활용하여 스케줄 기반의 처리와 실시간 처리의 기능을 동시에 지원.

BDD(Business Driven Development)의 기간에 맞추어 CI/CD의 개발 환경에서 DEV->UAT->QA->PROD의 단계별 테스팅 환경을 통해 담당 테스터들과 소통하여 안정적이고 기능이 보장된 서비스를 배포.
